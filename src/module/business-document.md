# 业务资料

本项目详实登记一个企业的业务资料，规范化一个企业的业务实体与业务流转等。
作为软件设计阶段的输入资料。基于这些资料形成数据库设计和测试用的业务数据。

## 组织形式

资料初始的组织形式，按照组织、部门、岗位三层分类，并明确资料是哪个岗位产生，作为哪个岗位的输入（具体到数据元素层级）。

```txt
根节点
├── docs (文档)
│   ├── 组织1
│   │   ├── 部门1
│   │   │    ├── 业务单据1
│   │   │    │    ├── 文档1.docx (同一个单据的多个具体文档)
│   │   │    │    ├── 文档1.md 对文档1.docx 的原文、分析和归纳，跟该文件同名，后缀名为md
│   │   │    │    ├── 文档2.docx
│   │   │    │    ├── 文档2.md 记录是哪个角色使用的，哪位员工提供的，上游表单与下游表单(markdown文件之间用超链接连接)
│   │   │    │    ├── index.md 单据介绍
│   │   │    │    ├── ai-input.md - 生成设计时，对 AI 提的要求
│   │   │    │    ├── ai-output.md - AI 生成的设计，存储原始输出，人工整理后的结果放到 index.md 文件中
│   │   │    │    └── data - 从文档中提取出来的业务数据，一个场景一个csv文件，其中包括输入和期望的输出
│   │   │    │        ├── 业务场景1.csv
│   │   │    │        ├── 业务场景2.csv
│   │   │    │        └── index.md 对业务场景和数据的介绍，一个业务场景的完成需要关联到多个表单，在此处描述业务场景并能通过超链接关联起来
│   │   │    ├── 业务单据2
│   │   │    └── index.md - 包括部门简介、部门下的岗位列表和文档列表，岗位与文档的读写汇总视图(代码计算)
│   │   ├── 部门2
│   │   └── index.md 组织简介，并罗列组织每个部门下的 index.md 清单
│   └── 组织2
├── mettings (会议记录)
│   ├── {yyyyMM}
│   │   ├── {dd}{2位流水号，如01}
│   │   │   ├── index.md (会议记录，要包含对 audio、video和image的清单说明)
│   │   │   ├── audios
│   │   │   │   ├── 01.m4a (2位流水号，从01开始)
│   │   │   │   └── 02.m4a
│   │   │   ├── videos
│   │   │   └── images
│   │   └── {dd}{2位流水号，如01}
│   └── {yyyyMM}
├── scenario (业务场景)
│   ├── {场景英文单词，用中划线分割}.md，定义业务场景、环节、参与人、参与的软件系统、输入资料和输出资料等，逐步形成业务场景的表达规范
│   ├── {场景英文单词，用中划线分割}.md，要跟单据data中的数据关联起来
│   ├── index.md 业务场景清单
└── docs.json 项目配置
```

文档目录（编写一段代码，进行统计等）

## 内容格式

1. Word 文档，docx 后缀
2. Excel 文档，xlsx 后缀
3. PDF 文档，pdf 后缀
4. 图片
5. 会议记录，包括 markdown、音频和照片等

## 分析过程

分析方包括：
1. 企业方的业务专家
2. 软件开发方的业务专家
3. 生成式 AI
